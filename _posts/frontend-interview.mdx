## 네트워크

- ### Http와 Https의 차이에 대해서 설명해 주세요

  `Http`는 **Hyper Text Transfer protocol**의 약자입니다. 웹에서 이루어지는 모든 데이터 교환의 기초가 되며,
  주로 HTML 문서나 이미지와 같은 리소스를 전송할 때 사용됩니다. 클라이언트가 서버에 요청을 보내면 서버는 해당 요청에 대한 응답을 보내줍니다.

  `Https`는 http에 데이터 암호화를 추가한 버전입니다. `Http`는 기본적으로 데이터를 평문으로 전송하기 때문에,
  제 3자가 네트워크 트래픽을 감시하여 사용자의 정보를 쉽게 도청할 수 있습니다. 이러한 취약점을 보안하기 위해서 `Https`가 도입되었습니다. 주요 차이점은 다음과 같습니다.

  1. 첫 번째, `SSL` 또는 `TLS` 프로토콜을 사용하여 데이터를 암호화 합니다.
  2. 두 번째, 서버의 신원을 확인할 수 있는 디지털 인증서를 사용합니다. 웹 브라우저 주소창에 있는 자물쇠 버튼을 클릭하면 인증서를 확인할 수 있습니다.
  3. 세 번째, 데이터가 전송 중에 변경되지 않는것을 보장합니다. 만약 암호화 과정에서 데이터가 변경되면, 수신자 측에서 이를 감지할 수 있습니다.

  이러한 이유로 웹상의 많은 사이트들이 `Https`를 기본 프로토콜로 채택하고 있습니다.

- ### CORS 문제를 겪어보신 적이 있으신가요? CORS가 무엇인지, 그리고 해결방법에 대해 설명해 주세요.

  `CORS(Cross-Origin Resource Sharing)`는 사실상 `동일 출처 정책(Same-Origin Policy)`으로 인한 제한을 해결하기 위한 방법입니다.
  동일 출처 정책은 웹 보안의 핵심 원칙 중 하나로, 프로토콜, 도메인, 포트가 모두 같은 경우에만 리소스가 다른 리소스와 상호작용할 수 있도록
  제한합니다. 이 정책은 `XSS`나 `CSRF`와 같은 온라인 공격으로부터 사용자를 보호하는 데 필수적입니다.

  그러나, 현대의 웹 애플리케이션은 다양한 출처의 리소스를 자유롭게 사용하고 상호작용하는 복잡한 구조로 발전했습니다.
  이런 상황에서 `CORS`는 다른 출처의 리소스를 안전하게 요청하고 사용할 수 있는 방법을 제공합니다. CORS는 서버가 특정 출처에서의 요청을 허용하도록 설정할 수 있게 하며,
  이는 서버가 `Access-Control-Allow-Origin` 같은 HTTP 헤더를 통해 어떤 출처의 요청을 수락할지 명시함으로써 이루어집니다.

  결론적으로, CORS는 웹의 보안 모델을 유지하면서도 다양한 출처의 리소스가 서로 상호작용할 수 있도록 하는 중요한 메커니즘입니다.
  이를 통해 개발자는 보안을 해치지 않으면서도 보다 풍부하고 상호 연결된 웹 경험을 제공할 수 있게 됩니다.

## 브라우저

- ### 로컬스토리지 vs 세션스토리지 vs 쿠키를 설명해주세요.

  #### 웹 스토리지: 로컬 스토리지와 세션 스토리지

  **로컬 스토리지(Local Storage)**: 웹 애플리케이션의 데이터를 장기간 저장하기 위해 사용됩니다.
  사용자가 브라우저를 닫더라도 데이터는 유지됩니다. 대부분의 브라우저에서는 최대 5MB까지 데이터를 저장할 수 있으며,
  같은 도메인 내의 모든 페이지에서 접근 가능합니다. 사용자의 선호 설정이나 테마 선택 같은 정보를 저장하는 데 주로 사용됩니다.

  **세션 스토리지(Session Storage)**: 한 세션 동안, 즉 브라우저 탭이 열려 있는 동안에만 데이터를 저장합니다.
  탭을 닫으면 저장된 데이터는 사라집니다. 로컬 스토리지와 마찬가지로 최대 5MB의 데이터를 저장할 수 있으며,
  저장된 데이터는 같은 탭 내에서만 접근 가능합니다. 폼 데이터 저장이나 사용자의 세션별 상태 관리에 사용됩니다.

  #### 쿠키 (Cookies)

  - 쿠키는 사용자의 세션 관리, 사이트 선호 설정, 사용자 추적 등에 사용됩니다.
    쿠키는 서버와 클라이언트 간의 요청과 응답 시 자동으로 포함되어 데이터를 주고받을 수 있습니다.
    저장 용량은 쿠키 당 4KB로 매우 제한적입니다. 쿠키는 도메인과 경로에 따라 접근성이 제한되며,
    서버로 전송되므로 서버에서도 접근할 수 있습니다. 사용자 인증, 세션 관리, 사용자 행동 추적(Google Analytics 등)에 주로 사용됩니다.

## JavaScript

- ### 클로저

  클로저는 함수가 자신이 생성될 때의 환경을 참조하고 유지하는 프로그래밍 개념입니다.
  간단히 말해서, 함수가 정의될 때 주변에 있던 변수와 스코프를 '기억'하고,
  이러한 정보를 기반으로 나중에 실행될 때도 해당 환경의 데이터에 접근할 수 있게 해줍니다.
  이 기능 덕분에, 함수는 실행될 때마다 그 환경의 변수들을 활용할 수 있으며, 이는 함수가 특정 계산 결과를
  '기억'하고 재사용할 수 있게 만들어줍니다.

  상태 관리 측면에서도 클로저는 중요한 역할을 합니다. 예를 들어, React와 같은 라이브러리에서는
  `useState`나 `useEffect` 같은 Hook을 사용하여 컴포넌트의 상태와 생명주기를 관리합니다.
  클로저는 이러한 Hook들이 컴포넌트의 이전 상태 값을 '기억'하고, 이를 기반으로 상태 업데이트나 연산을 수행할 수 있게 해줍니다.

  결론적으로, 클로저는 프로그래밍에서 함수가 선언된 시점의 렉시컬 환경에 대한 접근을 가능하게 하여,
  데이터 은닉, 상태 관리, 콜백 함수, 고차 함수 등 다양한 패턴과 기능을 구현하는 데 필수적인 역할을 합니다.
  이러한 특성을 통해 개발자는 더욱 유연하고 효율적인 코드를 작성할 수 있게 됩니다.
